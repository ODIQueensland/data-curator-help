Version

http://specs.frictionlessdata.io/data-package/#version

It is recommended that data package version follow the semanitic versioning approach, i.e. Given a version number Major.Minor.Patch, increment the:

  • Major version when either the general form of the data being produced changes or there is a change in the data source.
  • Minor version when anything else that touches the data changes E.g., changes in data location, corrections to the missingValues field, etc.
  • Patch version when there are minor metadata changes.
