Set Column Properties

Describe each Column so others can understand how they can use it. Each Column must have a name. It is good practice to provide a title and description.

Set column properties

  • Choose Tools > Set Column Properties, from the menu to display the Column panel
  • Enter properties to describe the Column
  • Column properties are saved when you export the data package

Required properties

  • Name - a short url-usable (and preferably human-readable) name of the table. This MUST be lower-case and contain only alphanumeric characters along with ".", "_" or "-" characters.

Recommended properties

  • Title - a title for the column.
  • Description - a summary of information in the column.

Optional properties

  • Type and Formats define the data type (e.g. string, integer, date) and its format (e.g. a string could be an email or a url)
  • Constraints limit the values that can be used in the field e.g. it must be unique, required, or less than a maximum value.
  • RDF Type is a way to give a rich description of the data. The RDF Type must be a URL of a RDF Class, that is an instance or subclass of RDF Schema Class object, e.g. http://schema.org/Country.
