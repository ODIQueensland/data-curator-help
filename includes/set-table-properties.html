Set Table properties

Describe the Table so others can understand how they can use it. The Table must have a name. It is good practice to provide a title and description.

Set table properties

  • Choose Tools > Set Table Properties, from the menu to display the Data Package panel
  • Enter properties to describe the Data Package
  • Data Package properties are saved when you export the data package

Required properties

  • Name - a short url-usable (and preferably human-readable) name of the table. This MUST be lower-case and contain only alphanumeric characters along with ".", "_" or "-" characters.

Recommended properties

  • Title - a title for the table.
  • Description - a summary of information in the table.

Optional properties

  • Sources - the raw sources for this data package
    • Move the pointer over the Add source button and click it to show the data entry boxes.
    • Enter values
    • Click the Add source button again to add another source
    • Click the Delete button to remove a source
  • Licenses help people understand how they can use the data. Licence is inherited from the Data Package unless you specify it here.
    • Move the pointer over the Licence you wish to set as a default and click it to select it.
    • Hold down Ctrl/Cmd to select more than one Licence.
    • The Data Package Licence is inherited by all the tables in the package unless you explictly apply a licence to a table using Set Table Properties.
