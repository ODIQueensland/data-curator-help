Restructure data in a data package

After you have created a data package, and perhaps updated the data in the data package over time, there may be some reason why you now need to restructure the data in the data package. This will impact the users of the data. You need to inform users of the changes and help them to process the new data. This will involve:

  • creating a new schema
  • validation
  • new provenance
  • new data package with version change
  • Open the existing data package table
  • Open new data, select all, copy data
  • Go to existing data tab, select all, delete, paste
  • Delete new data tab
  • Guess column properties or edit column properties
  • Validate table to check the new data complies with the column properties
  • Set table properties to update the description of the table
  • Set provenance information
  • Set data package properties to update the version o
  • Export data package to share the new data with others
